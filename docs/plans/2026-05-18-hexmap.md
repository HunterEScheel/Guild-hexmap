# D&D Interactive Hexmap Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use executing-plans to implement this plan task-by-task.

**Ticket:** none (personal project)

**Goal:** Build an interactive D&D hexmap where admins assign terrain and quests to hexes, and players can view the map and join quests in real-time.

**Architecture:** React SPA with SVG hex grid rendering. Firebase Firestore for real-time shared state. No auth — players enter names on first quest join, admin unlocks via PIN. Flat-top hex grid with offset coordinates, auto-scaling to 1 hex beyond filled bounds (minimum 40x20).

**Tech Stack:** React 18, TypeScript 5, Vite, Firebase (Firestore), SVG

**Standards:** React/TypeScript standards from tech-standards-devs (strict mode, feature-based structure, camelCase functions, PascalCase components)

**What to do:**

`src/types/index.ts` — Define all shared types:

- `QuestLevel`: union type — `"explore" | "recurring" | "wolf" | "demon" | "dragon" | "terrasque" | "god"`

`src/utils/colors.ts` — Define color maps:
- `TERRAIN_COLORS`: Record<TerrainType, string> mapping each terrain to a hex color (forest=#2d5a1e, plains=#7ec850, mountain=#8a8a8a, swamp=#4a6741, desert=#d4a46a, snow=#e8e8f0, water=#3a7bd5, unknown=#555555)
- `QUEST_LEVEL_COLORS`: Record<QuestLevel, string> mapping each quest level to a glow color (explore=#4ade80 green, recurring=#60a5fa blue, wolf=#facc15 yellow, demon=#f97316 orange, dragon=#ef4444 red, terrasque=#a855f7 purple, god=#fbbf24 gold)
- `QUEST_LEVEL_LABELS`: Record<QuestLevel, string> for display names

## Task 3: Hex Math Utilities

**Files:**
- Create: `src/utils/hexMath.ts`
- Create: `src/hooks/useGridSize.ts`

**What to do:**

`src/utils/hexMath.ts` — Flat-top hex coordinate math:
- `HEX_SIZE` constant (default 30px radius)
- `hexToPixel(col, row)` — convert offset coordinates to pixel center. Flat-top hex formula: x = col * 1.5 * size, y = row * sqrt(3) * size + (col % 2 === 1 ? sqrt(3)/2 * size : 0)
- `hexCorners(centerX, centerY, size)` — return 6 corner points for SVG polygon
- `hexPointsString(col, row)` — return SVG points attribute string for a hex at (col, row)

`src/hooks/useGridSize.ts` — Auto-scaling grid hook:
- Input: map of hex data (all non-unknown hexes)
- Calculate bounding box of filled hexes
- Extend 1 hex in each direction (minCol-1, maxCol+1, minRow-1, maxRow+1)
- Clamp to minimum 40 columns x 20 rows
- Return `{ minCol, maxCol, minRow, maxRow, totalCols, totalRows }`

**Commit:** `Add hex math utilities and grid sizing hook`

- Read config from environment variables (`VITE_FIREBASE_*`)
- Initialize and export Firestore instance
- Collections: `hexes` (doc ID = `{col}_{row}`), `quests` (auto-generated IDs), `config` (single doc `admin` with hashed PIN)

`src/hooks/useFirebase.ts` — Real-time data hooks:
- `useHexData()` — subscribe to `hexes` collection with `onSnapshot`, return `Map<string, HexData>` keyed by `{col}_{row}`
- `useQuests()` — subscribe to `quests` collection, return `Quest[]`
- `setHexTerrain(col, row, terrain)` — upsert hex document
- `createQuest(quest)` — add quest document
- `updateQuest(id, updates)` — partial update quest
- `deleteQuest(id)` — remove quest
- `joinQuest(questId, playerName)` — add name to players array (use `arrayUnion`)
- `leaveQuest(questId, playerName)` — remove name from players array (use `arrayRemove`)

`.env.example` — Document all required `VITE_FIREBASE_*` variables
